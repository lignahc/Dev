NAME

column - 목록을 가로 정열  

SYNOPSIS

column [-tx ] [-c columns ] [-s sep ] [file ... ]  

DESCRIPTION

column 명령은 입력된 것을 일정 칸으로 나누어 나열한다. 흡사 ls(1) 명령을 아무 옵션 없이 실행시켰을 때와 같은 기능을 한다. 먼저 세로로 나열하고 그 다음 가로로 나열한다. file 인자로 입력을 받거나, 일반적으로 표준 입력에서 입력을 받는다. 입력되는 것 중에 빈줄은 무시된다.

사용할 수 있는 옵션은 다음과 같다:

-c
전체 가로 폭을 columns 값으로 해서 정열한다.
-s
-t 옵션을 위해 가로칸 조사에 사용될 문자를 지정한다.
-t
입력되는 내용의 가로칸 갯수를 조사하고, 그것을 바탕으로 출력 양식을 정한다. 입력되는 내용의 가로칸 갯수는 공백문자로 구분한다. (초기값) 이 구분 문자는 -s 옵션에서 따로 지정할 수도 있다. 예쁜 출력물을 얻는데, 유용하게 쓰인다. (아래 예제 참조)
-x
가로로 먼저 나열하고, 다음 세로로 나열한다.

Column 명령은 오류 없이 끝나면 0, 오류가 발생하면, 0보다 큰 값을 리턴한다.  

ENVIRONMENT

COLUMNS
이 환경 변수는 가로칸의 기본 넓이를 지정하는데 사용된다. (기본적으로 콘솔에서는 특별히 지정하지 않는 한은 80이 되겠지요?)

 

EXAMPLES

(printf "PERM LINKS OWNER GROUP SIZE MONTH DAY HH:MM/YEAR NAME\n" ; \
ls -l | sed 1d) | column -t  

SEE ALSO

colrm(1), ls(1), paste(1), sort(1)  

HISTORY

column 명령은 BSD 4.3 Reno 에서 처음 나타났다.
