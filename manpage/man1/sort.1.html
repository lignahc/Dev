이름

sort - 텍스트 파일을 줄단위로 정렬  

문법

sort [OPTION]... [FILE]...  

설명

FILE(s)의 조합을 정렬한 후 결과를 표준 출력으로 출력한다.

+POS1 [-POS2]
POS1에서 키를 시작하고, POS2 전에서 끝낸다. (구식 옵션) 필드 번호와 문자 오프셋은 0으로 시작하는 숫자이다. (이것은 -k 옵션과 대조된다.)
-b
필드나 키를 정열할 때 라인 앞부분의 공백을 무시한다.
-c
이미 정렬된 상태인지를 검사하고, 정렬된 상태라면 정렬하지 않는다.
-d
키 값에서 오직 [a-zA-Z0-9 ] 문자들만 고려한다.
-f
키 값의 소문자를 대문자 사이에 끼워넣는다.
-g
일반적인 숫자 값에 따라 비교한다. -b 옵션을 내포한다.
-i
키 값에서 [\040-\0176], 즉 프린트할 수 있는 문자들만 고려한다.
-k POS1[,POS2]
POS1에서 키를 시작하고, POS2에서 끝낸다. 필드 번호와 문자 오스펫은 1로 시작하는 숫자이다. (이것은 0으로 시작하는 +POS 형태와 대조된다.)
-m
이미 정렬되어 있는 파일들을 합친다. 이때 다시 정렬하는 것은 아니다.
-M
(unknown) < `JAN' < ... < `DEC' 관계를 비교한다. -b 옵션을 내포한다.
-n
문자열 숫자값에 따라 비교한다. -b 옵션을 내포한다.
-o FILE
결과를 표준 출력이 아닌 지정된 FILE로 출력한다.
-r
비교 결과를 역으로 뒤집는다. (내림차순 정렬을 의미)
-s
마지막 재정렬 비교를 하지 못하도록 정렬을 안정화시킨다.
-t SEP
공백 문자(빈칸, 탭문자 등) 대신 필드 구분자로 지정된 SEP를 사용한다.
-T DIRECTORY
$TMPDIR이나 /tmp 대신 지정된 DIRECTORY를 임시 디렉토리로 사용한다.
-u
-c 옵션과 함께 사용할 경우, 정확한 정렬을 검사한다. 단독으로 사용된 경우, 동일한 실행의 처음만 출력한다.
-z
개행문자가 아닌 0 바이트로 행을 끝낸다. find -print0 에서 사용할 수 있다.
--help
도움말을 출력하고 종료한다.
--version
버전 정보를 출력하고 종료한다.

POS는 F[.C][OPTS] 의 형태를 가지며, 여기에서 F는 필드 번호(컬럼 번호), C는 필드에서 글자 위치를 나타낸다. 이 둘은 -k 옵션에서는 1부터 세며, 구식 옵션에서는 0부터 센다. OPTS는 하나 혹은 그 이상의 Mbdfinr를 설정한다. 이것은 그 키에 대한 전역 -Mbdfinr 설정을 비활성화시킨다. 만약 키가 주어지지 않으면, 전체 줄을 키로 사용한다. 만약 FILE이 주어지지 않거나 FILE이 -로 주어진 경우, 표준 입력으로부터 읽어들인다.

관련 항목

sort 에 대한 전체 문서는 Texinfo 메뉴얼로 유지된다. 여러분의 사이트에 info 와 sort 가 설치되어 있다면,
info sort

명령을 사용하여 완전한 메뉴얼을 볼 수 있다.
