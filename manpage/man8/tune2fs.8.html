이름

tune2fs - 개정판 확장 화일시스템의 파라미터를 튜닝한다.  

개용

tune2fs [ -l ] [ -c 최대 마운트 횟수 ] [ -e 에러 발생시 반응 ] [ -i 각 점검 간의 간격 ] [ -m 예약 블럭 퍼센트 ] [ -r 예약 블럭 갯수 ] [ -u 사용자 ] [ -g 그룹 ] 장치명  

설명

tune2fs 은 리눅스 개정판 확장 화일시스템의 몇몇 변경가능한 파라미터값을 설정한다.

파라미터 변경시 읽기/쓰기 마운트된 화일 시스템에 tune2fs 를 절대 사용하지 말라.

 

옵션

-c 최대 마운트 횟수
각 화일 시스템 점검 사이에 최대 몇 번의 마운트가 가능한지 설정
-e 에러 발생시 반응
에러가 발생했을 때의 커널 코드의 반응양식을 변화시킨다. errors-behavior 값은 다음 값을 가질 수 있다:
continue 정상적으로 실행을 계속한다.
remount-ro 읽기 전용으로 다시 마운트한다.
panic 커널 패닉을 일으킨다.
-g 그룹
예약된 블럭을 사용할 수 있는 사용자 그룹을 설정한다.
그룹 값은 GID 또는 그룹명이 올 수 있다.
-i 각 점검 간의 간격[d|m|w]
화일 시스템 점검 간의 최대 시간 간격을 설정한다. 접미어를 두지 않거나 `d'는 날짜, `m'은 달, `w'은 주를 뜻한다. 0 값은 시간 점검을 하지 않게 한다.
-l
화일 시스템의 수퍼블럭 내용을 나열한다.
-m 예약 블럭 퍼센트
주어진 장치의 예약 블럭 퍼센트를 조절한다.
-r 예약 블럭 갯수
주어진 장치 상의 예약 블럭 갯수를 조절한다.
-u 사용자
예약 블럭을 사용할 수 있는 사용자를 설정한다. 사용자 는 UID 또는 사용자명이 될 수 있다.

 

버그

아직 버그를 찾지 못했다. 버그가 있을 수는 있으나 있을 것 같지 않다.

 

주의

이 유틸리티는 여러분의 책임 하에 실행하기 바란다. 여려분은 화일 시스템을 수정하고 있는 것이다.  

저자

tune2fs 은 ext2 화일시스템의 개발자이자 관리책임자인 Remy Card <card@masi.ibp.fr>씨가 만들었다.
tune2fs 은 Theodore Ts'o <tytso@mit.edu>씨가 만든 ext2fs 라이브러리를 사용한다.
이 맨페지이즌 Christian Kuhtz <chk@data-hh.Hanse.DE>씨가 작성하였다.
시간에 의한 점검은 Uwe Ohse <uwe@tirka.gun.de>씨가 추가하였다.  

구할 수 있는 곳

tune2fs 은 ftp.ibp.fr 또는 tsx-11.mit.edu 에서 익명 ftp 를 통하여 /pub/linux/packages/ext2fs 디렉토리에서 구할 수 있다.
